# unimsg / UniMessage

用于描述大部分社交平台之间共通的基本消息内容的 **统一消息格式**。

UniMessage 是为了能够在框架中维护一套基本的平台无关的消息处理逻辑而存在的。

每个平台在消息的数据结构上可能会有很大的差异，但大部分社交平台的消息内容都至少包含为以下几种类型：

- 文本 （Text）
- 图片 （Image）
- 语音 （Audio）
- 视频 （Video）
- 文件 （File）
- 提及 （Mention）
- 引用 （Quote）

这使得统一的中间消息格式的存在是可行的，并且能够在大部分情况下满足跨平台消息处理的需求。

对于不需要用到一个平台专有的消息内容的功能来说，使用 UniMessage 作为消息处理的中间格式可以大大提升插件的跨平台兼容性，使得一个插件同时以复数的平台作为目标完全是可能的。

适配器应该实现将 UniMessage 转换为平台专有消息格式的逻辑，反之亦然，同时为了方便迁移，适配器在实现对应平台的消息处理逻辑时应该尽量和 UniMessage 保持相似，尽可能继承并实现 UniMessage 的接口。
